Endpoints

Custom REST APIs

Four custom endpoints for programmatic workflow control.

POST
/api/workflows/trigger
Manually trigger a workflow on any document. Requires workflowId, collectionSlug, and documentId.
GET
/api/workflows/status/:docId
Get complete workflow status, step progress, and audit trail for a document.
POST
/api/workflows/action
Approve, reject, review, or comment on the current active step. Requires authentication.
POST
/api/workflows/escalate-sla
Manually trigger SLA escalation check across all in-progress workflow instances.
